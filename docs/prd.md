# 论坛剧场数字化产品需求文档（PRD）

**产品名称**：观演者（Spect-Actor）  
**产品定位**：AI 驱动的沉浸式社会议题探索平台  
**核心价值**：好用、好看、好玩的思想启发工具  
**文档版本**：v1.0  
**更新日期**：2026-03-07

---

## 一、产品概述

### 1.1 产品愿景

打造一个让用户在"玩"中思考、在"演"中成长的数字化论坛剧场，通过 AI 技术实现观众与虚拟角色的深度互动，探索社会结构性困境，提升公民的对话能力和民主素养。

### 1.2 设计原则

#### 好用（Usability）

- **零学习成本**：无需教程，直觉式交互
- **流畅体验**：每个环节衔接自然，无卡顿感
- **清晰反馈**：每个操作都有即时、明确的反馈
- **容错机制**：允许用户随时暂停、回顾、重来

#### 好看（Aesthetics）

- **电影级视觉**：采用电影化的镜头语言和视觉叙事
- **情绪化设计**：色彩、动效、音效随剧情情绪变化
- **角色可视化**：每个角色都有独特的视觉形象和动画
- **沉浸式界面**：最小化 UI 干扰，最大化内容沉浸

#### 好玩（Engagement）

- **即时反馈**：用户的每句话都能引发 AI 角色的真实反应
- **意外惊喜**：小丑的幽默吐槽、AI 的"背地里评价"
- **成就感**：通过"英雄类型"等趣味化标签增强认同感
- **社交传播**：精美的报告卡片，易于分享到社交媒体

### 1.3 目标用户

**核心用户**：25-45 岁，大专及以上学历，关注社会议题，寻求有意义的互动体验

**用户细分**：

- **思辨型**：喜欢深度思考，看重教育性
- **体验型**：喜欢新颖互动，看重沉浸感
- **社交型**：喜欢分享传播，看重话题性

## 二、产品架构

### 2.1 信息架构

```
首页
├── 议题广场（浏览所有剧本）
├── 我的参演（历史记录）
└── 关于论坛剧场（理念介绍）

单个剧本体验流程
├── 01. 议题介绍页
├── 02. 沉浸式观演（3 幕）
├── 03. 角色解构
├── 04. 小丑提问
├── 05. 选择介入点
├── 06. 沙盒对话
└── 07. 分析报告
```

### 2.2 技术架构

```
前端层
├── React + TypeScript
├── Framer Motion（动画）
├── Tailwind CSS（样式）
└── Zustand（状态管理）

AI 层
├── LLM API（角色对话生成）
├── 情感分析引擎（压力值/火药味计算）
├── 对话分析引擎（边界感/策略性/同情响应）
└── 报告生成引擎

数据层
├── 剧本数据库（剧本、角色、介入点）
├── 用户数据库（对话历史、报告记录）
└── 分析数据库（用户行为、对话质量）
```

## 三、核心功能详细设计

### 3.1 议题广场（首页）

#### 3.1.1 功能目标

- 让用户快速找到感兴趣的议题
- 传达产品的独特价值和理念
- 降低用户的尝试门槛

#### 3.1.2 界面设计

**布局**：

```
[顶部导航栏]
  Logo | 议题广场 | 我的参演 | 关于

[Hero 区域]
  标题：成为观演者，探索社会困境
  副标题：不是解决问题，而是理解问题
  [开始探索] 按钮

[议题卡片网格]
  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
  │ 卡片 1      │ │ 卡片 2      │ │ 卡片 3      │
  │ [封面图]    │ │ [封面图]    │ │ [封面图]    │
  │ 标题        │ │ 标题        │ │ 标题        │
  │ 简介        │ │ 简介        │ │ 简介        │
  │ 标签 | 时长 │ │ 标签 | 时长 │ │ 标签 | 时长 │
  └─────────────┘ └─────────────┘ └─────────────┘
```

**议题卡片设计**：

- **封面图**：情绪化的插画或摄影，传达议题氛围
- **标题**：简洁有力，如"城里的月光"
- **简介**：一句话概括核心冲突，如"当农村老人走进城市，家庭的裂痕开始显现"
- **标签**：议题分类，如"家庭关系"、"代际冲突"、"城市化"
- **时长**：预计体验时间，如"20-30 分钟"
- **参演人数**：显示已有多少人参演，增强社交证明

**交互设计**：

- 鼠标悬停时，卡片微微放大，封面图有视差效果
- 点击卡片，平滑过渡到议题介绍页

#### 3.1.3 技术实现

- 使用 CSS Grid 实现响应式卡片布局
- 使用 Framer Motion 实现悬停动画
- 封面图使用 WebP 格式优化加载速度

### 3.3 沉浸式观演（核心体验 1）

#### 3.3.1 功能目标

- 让用户对角色产生情感共鸣
- 让用户理解冲突如何逐步升级
- 为后续介入建立情感和认知基础

#### 3.3.2 界面设计

**布局**：

```
[全屏对话界面]

  [顶部进度条]
    第 1 幕 ━━━━━━●━━━━━━ 第 2 幕 ━━━━━━━━━━ 第 3 幕

  [左侧：情绪指标]
    压力值 ▲ 75%
    火药味 🔥 高

  [中央：对话区域]
    ┌────────────────────────────────┐
    │ [角色头像] 邱华                │
    │ "哎呀，我的大孙子呢？快让奶奶  │
    │  看看！"                       │
    └────────────────────────────────┘

    ┌────────────────────────────────┐
    │                    小雅 [角色头像] │
    │  "妈，您辛苦了，快进来坐。妈， │
    │   这鸡......城里不好养......"  │
    └────────────────────────────────┘

  [底部：操作区]
    [点击屏幕继续] 或 [向上滑动回顾]
```

**对话呈现方式**：

- **电影化布局**：对话气泡左右交替，模拟真实对话
- **角色头像**：每个角色有独特的插画头像，增强识别度
- **打字机效果**：文字逐字显示，增强代入感（可跳过）
- **语音朗读**（可选）：为每个角色配音，增强沉浸感
- **动态场景背景**：
  - 根据剧情场景切换背景图片（如客厅、卧室、厨房）
  - 背景使用模糊效果（blur）或半透明遮罩，确保对话清晰可读
  - 背景色调随情绪变化：平静场景使用冷色调，紧张场景使用暖色调
  - 支持视差滚动效果，增强空间层次感
  - 背景图片预加载，确保切换流畅无卡顿

**情绪指标设计**：

- **压力值**：0-100% 的进度条，随剧情变化
  - 0-30%：绿色（平静）
  - 31-60%：黄色（紧张）
  - 61-100%：红色（爆发）
- **火药味指数**：用火焰图标和文字表示
  - 低：💧 平静
  - 中：🔥 升温
  - 高：🔥🔥 爆发

**幕间过渡**：

- 每一幕结束后，屏幕淡出，显示幕间标题
- 例如："第二幕：陌生的城市，熟悉的牵挂"
- 配合舒缓的音效，给用户短暂的情绪缓冲

#### 3.3.3 交互设计

**推进剧情**：

- **点击屏幕任意位置**：显示下一条对话
- **空格键**：显示下一条对话（键盘快捷键）
- **自动播放**（可选）：每 3 秒自动显示下一条

**回顾对话**：

- **向上滑动**：进入回顾模式，显示历史对话
- **时间轴导航**：在回顾模式中，可以跳转到任意对话

**暂停退出**：

- **左上角 [暂停] 按钮**：弹出菜单
  - 继续观演
  - 回到首页
  - 调整设置（音量、语速等）

#### 3.3.4 技术实现

**对话数据结构**：

```typescript
interface Dialogue {
  id: string;
  act: 1 | 2 | 3; // 第几幕
  speaker: string; // 说话者
  content: string; // 对话内容
  emotion: "calm" | "tense" | "angry"; // 情绪
  stressLevel: number; // 压力值 0-100
  tensionLevel: "low" | "medium" | "high"; // 火药味
}
```

**情绪计算**：

- 使用 NLP 情感分析 API（如 OpenAI、Anthropic）
- 分析对话中的负面词汇、冲突性语句
- 实时更新压力值和火药味指数

**性能优化**：

- 预加载下一幕的对话数据
- 使用虚拟滚动优化长对话列表
- 对话气泡使用 CSS 动画，避免 JS 计算

### 3.4 角色解构（核心体验 2）

#### 3.4.1 功能目标

- 帮助用户从情绪共鸣转向理性分析
- 让用户理解每个角色的动机和压力
- 为后续对话提供策略思考的基础

#### 3.4.2 界面设计

**布局**：

```
[全屏卡片展示]

  [顶部标题]
    让我们重新认识这些角色

  [角色卡片轮播]
    ┌─────────────────────────────────┐
    │         [角色插画]              │
    │                                 │
    │         邱华，60 岁             │
    │         农村妇女                │
    │                                 │
    │  核心动机：为了儿子，想要帮忙   │
    │  隐秘压力：对老伴的思念，城市   │
    │            的格格不入           │
    │  权力等级：家庭中的低位，话语   │
    │            权被剥夺             │
    │  行为底线：强忍委屈，但当老伴   │

    └─────────────────────────────────┘
```

**角色卡片设计**：

- **角色插画**：风格化的插画，传达角色的性格和情绪
- **信息层级**：
  - 一级：角色名称、年龄、身份
  - 二级：核心动机、隐秘压力、权力等级、行为底线
- **视觉编码**：
  - 核心动机：用 💡 图标
  - 隐秘压力：用 😰 图标
  - 权力等级：用 ⚖️ 图标
  - 行为底线：用 🚧 图标

**卡片切换动画**：

- 左右滑动切换卡片
- 卡片切换时有 3D 翻转效果
- 支持键盘左右箭头切换

#### 3.4.3 技术实现

**角色数据结构**：

```typescript
interface Character {
  id: string;
  name: string;
  age: number;
  role: string;
  avatar: string; // 插画 URL
  coreMotivation: string; // 核心动机
  hiddenPressure: string; // 隐秘压力
  powerLevel: string; // 权力等级
  behaviorBoundary: string; // 行为底线
}
```

**卡片轮播**：

- 使用 Swiper.js 或 Framer Motion 实现
- 支持触摸滑动和键盘导航
- 卡片数量显示：1/4, 2/4, 3/4, 4/4

### 3.5 小丑提问（核心体验 3）

#### 3.5.1 功能目标

- 激活用户的深度思考
- 从旁观者转变为行动者
- 收集用户的初步想法，为 AI 对话提供参考

#### 3.5.2 界面设计

**布局**：

```
[全屏对话界面]

  [小丑角色登场动画]
    [小丑插画：穿西装的猫]

    "嘿，看完这出戏，有什么想法吗？"
    "让我来问你几个问题……"

  [问题 1]
    你觉得邱华在哪一刻彻底失去了抵抗力？

    [文本输入框]
    请输入你的想法...

    [下一题] 按钮

  [问题 2]
    你认为小雅真的是"恶婆婆"吗？

    [文本输入框]

    [下一题] 按钮

  [问题 3]
    如果是你，你会怎么做？

    [文本输入框]

    [完成] 按钮
```

**小丑角色设计**：

- **视觉形象**：穿西装的猫（幽默、中立、有点调侃）
- **动画效果**：小丑从屏幕边缘"跳"进来
- **语言风格**：轻松、幽默，但不失深度

**问题设计原则**：

- **问题 1**：关注关键转折点（事实层面）
- **问题 2**：挑战二元思维（认知层面）
- **问题 3**：促使行动思考（行为层面）

**输入框设计**：

- 多行文本框，支持 100-500 字
- 实时显示字数统计
- 支持 Markdown 格式（可选）

#### 3.5.3 交互设计

**问题导航**：

- 必须回答当前问题才能进入下一题
- 支持返回修改之前的答案
- 进度指示：1/3, 2/3, 3/3

**跳过机制**：

- 如果用户不想回答，可以点击 [跳过]
- 但至少需要回答 1 个问题才能继续

**小丑反馈**：

- 用户输入后，小丑给出简短的幽默回应
- 例如："有意思，继续……"

#### 3.5.4 技术实现

**问题数据结构**：

```typescript
interface Question {
  id: string;
  question: string;
  placeholder: string;
  minLength: number;
  maxLength: number;
}
```

**用户回答存储**：

- 存储到本地状态（Zustand）
- 后续在 AI 对话中作为上下文参考

- 让用户主动选择想要改变的时刻
- 增强用户的代入感和控制感
- 为沙盒对话设定起点

#### 3.6.2 界面设计

**布局**：

```
[全屏时间轴界面]

  [顶部标题]

            介入点 1

                        ↑
                        介入点 2

                                ↑
                                介入点 3

    第 4 幕 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━●━━━
                                            ↑

  [介入点详情卡片]（鼠标悬停或点击时显示）
    ┌─────────────────────────────────┐
    │ 介入点 1：小雅拒绝活鸡          │
    │                                 │
    │ 场景：小雅刚出院回家，邱华带来 │

    │                                 │
    │ 冲突：城乡文化差异              │
    │                                 │
    │ 考验：沟通能力，如何让双方都感 │

    └─────────────────────────────────┘
```

**时间轴设计**：

- 横向时间轴，从左到右展示剧情进展
- 介入点用醒目的标记（如红色圆点）
- 鼠标悬停时，标记放大，显示详情卡片

**介入点卡片设计**：

- 简洁描述场景、冲突、考验
- 用不同颜色区分不同类型的冲突
  - 绿色：沟通类
  - 蓝色：同理心类
  - 橙色：边界感类
  - 红色：系统性思维类

#### 3.6.3 交互设计

**选择介入点**：

- 点击时间轴上的介入点，显示详情卡片
- 点击卡片中的 [选择这个时刻]，进入沙盒对话

**推荐机制**（可选）：

- 基于用户在小丑提问中的回答，推荐最适合的介入点
- 在推荐的介入点上显示 ⭐ 标记

#### 3.6.4 技术实现

**介入点数据结构**：

```typescript
interface InterventionPoint {
  id: string;
  act: number;
  position: number; // 在时间轴上的位置（0-100）
  title: string;
  scene: string;
```

**功能**：

```typescript
class DialogueAnalyzer {
  async analyzeDialogue(
    userInput: string,
    aiResponse: string,
    dialogueHistory: Message[],
  ): Promise<DialogueAnalysis> {
    // 1. 情感分析
    const sentiment = await this.analyzeSentiment(userInput);

    // 2. 策略分析
    const strategy = await this.analyzeStrategy(userInput, dialogueHistory);

    // 3. 边界感分析
    const boundary = await this.analyzeBoundary(userInput, dialogueHistory);

    // 4. 同情响应分析
    const empathy = await this.analyzeEmpathy(userInput, aiResponse);

    return {
      sentiment,
      strategy,
      boundary,
      empathy,
      tensionLevel: this.calculateTensionLevel(sentiment, strategy),
    };
  }

  private async analyzeStrategy(
    userInput: string,
    dialogueHistory: Message[],
  ): Promise<number> {
    // 使用 LLM 分析用户是否在寻找解决方案
    const prompt = `
      分析以下对话，判断用户是否在寻找建设性的解决方案（而非单纯争吵）。
      给出 0-100 的得分。
      
      对话历史：${JSON.stringify(dialogueHistory)}
      用户输入：${userInput}
    `;

    const response = await this.callLLM(prompt);
    return parseInt(response.content);
  }
}
```

## 十、风险与应对

### 10.1 技术风险

**风险 1：AI 对话质量不稳定**

- **应对**：建立角色提示词库，持续优化
- **应对**：引入人工审核机制，过滤低质量回复
- **应对**：提供"重新生成"功能，让用户可以重试

**风险 2：性能问题**

- **应对**：使用 CDN 加速静态资源
- **应对**：优化 AI API 调用，使用缓存
- **应对**：使用流式传输，减少等待时间

**风险 3：数据安全**

- **应对**：使用 HTTPS 加密传输
- **应对**：敏感数据加密存储
- **应对**：定期备份数据

### 10.2 产品风险

**风险 1：用户参与度低**

- **应对**：优化剧本质量，选择更贴近用户的议题
- **应对**：降低参与门槛，简化操作流程
- **应对**：增加趣味性元素，如小丑的幽默、报告的趣味化

**风险 2：用户留存率低**

- **应对**：定期更新新剧本，保持新鲜感
- **应对**：建立用户社区，增强社交连接
- **应对**：设置成就系统，鼓励用户体验更多剧本

**风险 3：付费转化率低**

- **应对**：提供高质量的免费体验，建立信任
- **应对**：设置合理的定价，降低付费门槛
- **应对**：提供会员专属内容，增强付费价值

### 10.3 运营风险

**风险 1：内容审核**

- **应对**：避免敏感政治话题
- **应对**：建立内容审核机制
- **应对**：提供用户举报功能

**风险 2：市场竞争**

- **应对**：强化产品差异化（论坛剧场理念、AI 即兴演出）
- **应对**：建立内容壁垒（高质量剧本）
- **应对**：建立社区壁垒（用户社区）
