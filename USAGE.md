# 使用指南

## 快速开始

### 1. 获取 Kimi API Key

1. 访问 [Kimi 开放平台](https://platform.moonshot.cn/)
2. 注册账号并登录
3. 在控制台创建 API Key
4. 复制 API Key

### 2. 配置环境变量

```bash
cp .env.local.example .env.local
```

编辑 `.env.local`：

```env
MOONSHOT_API_KEY=your_api_key_here
```

### 3. 启动应用

```bash
npm install
npm run dev
```

访问 http://localhost:3000

## 用户体验流程

### 第一步：选择剧本

在首页（议题广场）浏览可用的剧本，点击卡片进入剧本详情。

### 第二步：了解剧本

查看剧本的背景故事、角色信息和主题标签，点击"开始体验"。

### 第三步：沉浸式观演

自动播放剧本对话，观察角色情绪变化和压力值上升。可以：
- 点击"播放/暂停"控制播放
- 点击"查看角色信息"了解角色背景
- 点击"跳过观演"直接进入下一步

### 第四步：角色解构

深入了解每个角色的：
- 背景故事
- 核心动机
- 隐藏压力
- 权力水平
- 行为边界
- 语言风格

### 第五步：小丑提问

回答 3 个思辨问题：
1. 你觉得这个冲突的核心问题是什么？
2. 如果你是其中一个角色，你会有什么感受？
3. 你认为有什么更好的解决方案吗？

你的回答会被传递给 AI，影响后续对话。

### 第六步：选择介入点

选择一个关键时刻介入对话，每个介入点有不同的：
- 场景描述
- 冲突类型
- 挑战维度（沟通能力/同理心/边界感/系统性思维）

### 第七步：沙盒对话

与 AI 驱动的角色进行真实对话：
- 选择对话对象（邱华/小雅/阿强）
- 输入你的回应
- 观察 AI 角色的反应和情绪变化
- 系统会实时分析你的沟通能力
- 最多进行 30 轮对话

### 第八步：查看报告

获得个性化的沟通分析报告：
- **英雄类型**: 8 种类型之一（和平主义小白鸽、硬核边界守卫者等）
- **三维度评分**: 边界感、策略性、同理心
- **关键时刻**: 你最有代表性的一句话
- **AI 内心独白**: 角色对你的真实想法
- **社会学知识**: 针对你最弱维度的建议

## 技术说明

### AI 对话机制

- 使用 Kimi K2.5 模型
- 每个角色有独特的提示词（包含背景、动机、压力等）
- 支持上下文理解（包含用户在小丑提问环节的回答）
- 自动检测情绪（calm/tense/angry）

### 对话分析机制

- 使用 AI 分析每轮对话
- 评估三个维度：边界感、策略性、同理心
- 检测对话僵局（重复模式）
- 生成实时反馈

### 报告生成机制

- 基于所有对话的平均分数
- 8 种英雄类型判定规则
- 提取关键时刻（使用 AI）
- 匹配社会学知识

## 常见问题

### Q: 为什么对话没有响应？

A: 请检查：
1. `.env.local` 文件是否存在
2. `MOONSHOT_API_KEY` 是否正确配置
3. 网络连接是否正常
4. 浏览器控制台是否有错误信息

### Q: 如何重新开始？

A: 点击报告页面的"尝试其他介入点"或"返回首页"按钮。

### Q: 对话轮次用完了怎么办？

A: 对话会自动结束并跳转到报告页面。你可以选择其他介入点重新开始。

### Q: 如何添加新的剧本？

A: 在 `data/scripts/` 目录下创建新的 JSON 文件，参考 `city-moonlight.json` 的格式。

## 成本控制

### Kimi API 定价

- kimi-k2.5 模型：¥12/百万 tokens（输入）+ ¥12/百万 tokens（输出）
- 每轮对话约消耗 500-1000 tokens
- 30 轮对话约消耗 ¥0.18-0.36

### 优化建议

1. 限制对话轮次（默认 30 轮）
2. 限制单次回复长度（max_tokens: 200）
3. 使用缓存减少重复请求
4. 在生产环境添加用户配额限制

## 开发建议

### 添加新剧本

1. 创建 JSON 文件（参考 `city-moonlight.json`）
2. 定义角色（至少 2 个）
3. 编写对话（至少 3 幕）
4. 设置介入点（至少 2 个）
5. 在 `data/scripts/index.ts` 中导入

### 自定义英雄类型

编辑 `lib/types/report.ts` 中的 `HERO_TYPES` 常量，添加新的类型定义。

### 修改分析维度

编辑 `lib/engines/dialogue-analyzer.ts` 中的分析逻辑，调整评分标准。
